Battleships - An interpretation

 

What actually are Battleships?

Battle ships is a "strategy type guessing game intended for 2 players" - Wikipedia. It's thought to originate from the French game L'Attaque which was played by French soldiers during World War 1. It is most commonly played on a 8 x 8 grid, where each player places their ships on one board of their own, and then places guesses on their second board, which represent the opponents board. Each player has the following set of ships to place:

After both players prepare their boards, they then take turns trying to sink each others ships by calling out coordinates on the board. A round of Battleships would look something like this:

[Player 1] I'm firing at E3
[Player 2] Miss.
[Player 2] I'm firing at D7
[Player 1] Hit, but not sank.

 

The game continues like this until a player sinks all of his opponents ships.

 

Compromises in my code

From the beginning of coding this game, I knew that I'll have to make some compromises (at least for now) in order to make a fully functional game which would be enjoyable to play for anyone. One of the biggest decisions I made while coding was deciding to make the game single player. In a normal game, each player choses their own ships and then fight each other, however in my interpretation, the game places all of the ships and the player has to guess all of the ships.

Another compromise I made (not nearly as big as the previous one) was that the game will not tell the player if the ship has been sank. In the normal game, for example if I just finished firing at the battleship, my opponent would tell me that it sunk, so I know that there are no more possible combinations of 4 spaces where the battleship could be located. The reason why I haven't implemented this feature into my game was simple due to the fact that I couldn't find a simple and clean way of implementing the feature. I did try to implement it, however I noticed that the game became a lot more sluggish due to extra cycles of checking the whole grid for the number of 4's. However, to make up for this compromise, the player doesn't have a limited number of tries to sink all of the ships.

Some may think that the game isn't as enjoyable with this feature, but after some simple math, I realised that if I set the number of tries to 3 times the amount of ships, the player would have to guess 1 part of a ship per 3 turns, which is reasonable, but quite difficult due to the size of the grid.

 

Program logic - Breakdown of the game

Before even thinking of starting to program the game itself, I knew that I had to break down the game into chunks which would help me to visualise the code and think of possible ways of writing it. This process is called decomposition and it is used by all programmers for all sorts of tasks - game programming, simulations etc. This is how I decomposed the logic behind the game Battleships:

Setting up the grid - An empty grid with the dimensions of 8 by 8 must be created.

Populating the grid - All of the ships must be placed onto the board so that they are not overlapping.

Main game loop:

  • Display players guessing grid - this allows the player to keep track of his hits and misses.
  • Player input - The player must enter a coordinate in the format {A3}, which is then validated if there is such a coordinate, if the user entered it in the correct format and if the player has already guessed that specific coordinate before.
  • Input execution - the valid coordinate is checked against the grid where all of the ships are placed, and then if there is a ship located at that location, the players grid is updated with a "hit", and if not, it's updated with a "miss"
  • End game check - if all of the ships are sank, the main game loop finishes and the layer is presented with a win screen.

After a couple of hours of trying to visualise all of the above, I finally sat down and started to program in C#. I'm well aware that before staring to code in C# I should've written the whole program in pseudo code that would help me with all of the logic behind all of the steps in my decomposition, but I was feeling confident in myself and went straight into programming - which was quite a big mistake later on.

When all of the ships have been hit, the user is presented with a message You won! as well as the ship grid is displayed showing the user the full formation of the ships on the grid.
